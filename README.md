# 贝塔驿站微信小程序

贝塔驿站是面向校园生活场景的微信小程序，覆盖校园集市、树洞交流、内容发布、搜索、评论互动、个人中心和轻量工具等功能。项目使用微信小程序原生能力开发，并结合 Vant Weapp、WeUI 扩展库和自定义组件组织页面体验。

本文档用于介绍项目功能、目录结构、开发流程、截图规范和隐私边界。公开文档不包含 AppID、密钥、接口域名、token、真实用户数据或未脱敏截图。

## 项目定位

贝塔驿站的核心目标是提供校园内高频信息流转入口，让用户可以快速浏览、筛选、发布和管理校园内容。当前小程序包含三个主要入口：

- 集市：承载二手、失物、求助、兼职、租房等校园生活信息。
- 树洞：承载匿名或弱身份表达场景，包括吐槽、倾诉、心愿和讨论类内容。
- 个人中心：承载登录状态、校园认证、我的发布、我的评论、我的喜欢、回复通知、反馈和其他个人功能。

## 功能概览

### 集市信息流

- 支持多分类内容浏览、校区筛选、搜索入口和详情跳转。
- 支持图片预览、点赞、评论、价格或热度信息展示。
- 支持触底加载更多、加载态、骨架屏和基础过渡动画。
- 支持左右滑动切换主分类，并在边界处使用阻尼回弹。
- 横向滑动区域会裁剪越界位移，避免带动整页横向偏移。

### 树洞信息流

- 支持树洞分类浏览，包括全部、吐槽、倾诉、心愿等内容类型。
- 支持新发、新回、热门、精选等排序维度。
- 支持热榜入口、详情页、评论、点赞和回复互动。
- 支持树洞风格卡片、骨架屏、加载动画、列表过渡和滑动切换体验。

### 详情与评论

- 支持内容详情展示、图片预览、联系、返回和举报等操作入口。
- 支持评论列表、二级回复、点赞、删除和分页加载。
- 支持评论排序切换，以及评论区左右滑动切换排序。
- 滑动识别会避开点击、长按、图片预览和输入操作，避免影响现有交互。

### 内容发布

- 支持多类型内容发布、表单输入、图片上传和发布状态保护。
- 支持上传中局部状态，不使用遮挡全屏的加载提示。
- 支持发布过程中的重复提交防护。

### 个人中心

- 支持登录和用户状态入口。
- 支持校园认证、我的发布、我的评论、我的喜欢、我的回复。
- 支持客服、反馈、协议、申请等辅助入口。
- 支持轻量动画和按压反馈。

### 小工具

- 内置 2048 小游戏页面。
- 支持二维码相关页面。
- 支持 WebView 承载外部内容页面。

## 页面结构

- 首页：集市信息流、分类导航、校区筛选、搜索入口。
- 搜索页：内容搜索和搜索结果展示。
- 详情页：帖子详情、图片预览、评论和回复。
- 树洞页：树洞信息流、热榜入口、分类和排序。
- 热榜页：热门内容展示。
- 发布页：内容发布入口。
- 个人中心：首页式个人功能入口。
- 我的评论、我的喜欢、我的发布、我的回复：个人内容管理。
- 联系客服、意见反馈、登录页、认证申请页、协议页、二维码页、WebView 页和 2048 小游戏。

## 本地启动

1. 使用微信开发者工具导入当前小程序项目目录。
2. 确认小程序根目录指向项目内的 `miniprogram` 目录。
3. 安装 Node 依赖并在微信开发者工具中执行 npm 构建。
4. 选择合适的编译模式和页面路径。
5. 使用测试账号或安全的 mock 数据进行调试。

注意：README 不提供 AppID、接口域名、上传凭证或任何私密配置。开发者应通过本地安全配置或团队内部文档获取必要信息。

## UI 截图辅助

项目内置 `scripts/miniprogram-ui-capture.js`，用于连接微信开发者工具自动化服务并生成页面截图。脚本会把原始截图和页面数据写入本地忽略目录 `screenshots/miniprogram`，公开截图写入 `docs/assets/screenshots`。

推荐流程：

1. 开启微信开发者工具自动化服务，并信任当前项目。
2. 运行 `npm run ui:capture:contact` 抓取不包含用户内容的帮助与反馈页。
3. 运行 `npm run ui:capture:index` 或 `npm run ui:capture:treehole` 抓取信息流页面，脚本会对公开图进行基础区域打码。
4. 检查公开截图，确认头像、昵称、正文、评论和联系方式等用户信息已被遮盖。
5. 只提交 `docs/assets/screenshots` 中确认脱敏后的公开截图，不提交 `screenshots/miniprogram` 中的原始文件。

如果微信开发者工具已经使用固定端口启动，可在命令后追加 `-- --port 端口号`。不同版本的开发者工具对自动化端口参数支持不完全一致，当前脚本使用工具 CLI 支持的 `--port` 参数启动服务。

## 协作建议

- 修改页面前先确认是否影响 tabBar、导航栏、浮动按钮和滚动区域。
- 调整动效时只使用 transform 和 opacity，避免改变布局尺寸。
- 骨架屏应尽量复用真实卡片尺寸，避免加载态跳动。
- 上传、发布、评论等操作必须有提交中状态。
- 涉及用户内容的变更必须考虑内容审核和异常回退。
- 提交前建议至少完成页面预览、基础点击链路、触底加载、左右滑动和表单提交检查。
